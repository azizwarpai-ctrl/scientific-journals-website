## How to update this board

- Change `status` in the same PR that advances the task.
- If a task uncovers new scope, **add a new row** (T7+); do NOT widen an existing
  task. The new row links to the PR that surfaced it.
- `blocked-on-ops` means: repo work is complete (or N/A), waiting on a human
  operator confirmation. Do not progress past this status from the repo side.
- `done` requires: merged PR + human approval recorded + verification evidence
  attached.
