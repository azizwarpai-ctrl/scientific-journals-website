feat(email): implement complete event-driven email management system

This commit introduces the full Email Management System to digitopub, strictly adhering to the architectural constraint that OJS remains the sole identity provider. No local authentication, credential storage, or JWT session logic was introduced.

Key additions and changes:
1. Event-Trigger System:
   - Added `src/lib/email/events.ts` to define a typed event catalogue (e.g., `registration_confirmation`, `submission_received`).
   - Added `src/lib/email/event-dispatcher.ts` for fire-and-forget email dispatching mapped to these events.
   - Hooked up `dispatchEmailEvent` in the `POST /auth/register` route, ensuring emails are sent reactively after successful OJS provisioning without blocking the flow.

2. System Templates Seeding:
   - Created `scripts/seed-email-templates.ts` to idempotently seed 5 default OJS-event email templates (registration, submissions, reviews) using Prisma upsert.

3. Email Log Viewer & Admin UI Refactoring:
   - Added a `GET /logs` endpoint in the email-templates router, protected by `requireAdmin`, returning paginated delivery logs.
   - Added `useGetEmailLogs` hook and a fully featured `app/admin/email-templates/logs/page.tsx` displaying the delivery history with status badges and filtering.
   - Refactored `fetchStatus`, `handleDelete`, and `handleToggleActive` in the templates index page into reusable API hooks (`useGetEmailStatus`, `useDeleteEmailTemplate`, `useUpdateEmailTemplate`), aligning with project conventions.

4. Registration Flow Refinement:
   - Updated the Zustand registration store to track `selectedJournalPath`.
   - Fixed the post-registration redirect in `verify-code-form.tsx` to route users directly to the OJS SSO bridge (`/api/ojs/sso/redirect`), eliminating the incorrect redirect to the local login page.

5. Testing:
   - Wrote unit tests for the new `event-dispatcher.ts` ensuring resilient execution.
   - Validated that the existing 200+ unit and integration tests remain green.

Architectural Validation:
- [x] No local user credentials stored.
- [x] No new login endpoints created.
- [x] Email system remains purely event-driven and decoupled from Identity.
